## What is MDX?

**MDX** combines the simplicity of Markdown with the power of JSX components. This means you can write regular markdown **and** embed interactive React components directly in your blog posts!

<Callout type="info" title="Fun Fact">
This entire blog post is written in MDX! All the interactive elements you see are React components embedded in markdown.
</Callout>

## Why Use MDX for Technical Blogging?

Here are some compelling reasons:

1. **Interactive Code Examples** - Let readers experiment with code
2. **Custom Callouts** - Highlight important information
3. **Collapsible Sections** - Keep long content organized
4. **Reusable Components** - Build once, use everywhere

<Callout type="success" title="Pro Tip">
MDX is perfect for technical blogs because you can create custom components for diagrams, code playgrounds, and interactive visualizations!
</Callout>

### Under the Hood

The `CodePlayground` component uses React hooks:
- `useState` to manage the code content
- `navigator.clipboard` API for copy functionality
- Tailwind/UnoCSS classes for styling

Here's a simplified version:

```typescript
function CodePlayground({ code: initialCode }) {
  const [code, setCode] = useState(initialCode);

  const handleCopy = () => {
    navigator.clipboard.writeText(code);
  };

  return (
    <div>
      <button onClick={handleCopy}>Copy</button>
      <textarea value={code} onChange={(e) => setCode(e.target.value)} />
    </div>
  );
}
```
